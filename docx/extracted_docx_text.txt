Section 5: Troubleshooting
Tổng hợp các lỗi thường gặp trong quá trình xây dựng Agent và cách fix:
1. Account & Quyền truy cập (Access)
1.1. Nếu mình chưa có account Claude Teams/ Enterprise hay ChatGPT Plus thì có thể xây dựng được agent không?
Được. Nếu bạn chưa có các tính năng trả phí như Claude Cowork hay Claude Code, bạn vẫn có thể dùng Codex (gói miễn phí). Ngoài ra, nhiều IDE (phần mềm lập trình) như Visual Studio Code hoặc Antigravity cũng có extension cho phép chat và "vibe code" (lập trình bằng hội thoại với AI) với các model miễn phí.
2. Truy cập Repo Skills AgentBase (Git)
2.1. Repo báo private khi thực hiện lệnh git clone cho bộ skill của AgentBase
Lỗi này thường xảy ra khi máy bạn chưa cài Git CLI hoặc GitHub Desktop.  Cách khắc phục: bạn có thể tải bộ skill trực tiếp từ trang GitHub (dạng file nén) rồi đưa vào folder build Agent, thay vì dùng lệnh git clone. Tuy nhiên, bạn vẫn nên cài Git CLI hoặc GitHub Desktop, vì sau này cần đến chúng để đẩy (push) source code của agent lên GitHub.  Nếu dùng Claude Cowork, xem mục How to push repo to Github với Claude Cowork.
2.2. Lỗi "Git is required for local sessions" trong Claude Code
Nếu chưa cài Git CLI, khi mở Claude Code, chọn folder và import link bộ skill AgentBase, bạn sẽ thấy lỗi: "Git is required for local sessions". Để khắc phục, hãy cài Git CLI tại: https://git-scm.com/install/
3. Cấu trúc Folder để chứa Agent
3.1. Agent và bộ skill AgentBase cần được đưa vào một folder chung
Khi tạo agent, bạn cần một folder để chứa source code của agent. Bộ skill AgentBase cũng phải được import (hoặc git clone) vào chính folder đó. Nếu source code của agent và bộ skill AgentBase nằm ở hai folder khác nhau, quá trình build agent và chạy bộ skill có thể bị lỗi.
Gợi ý: khi bắt đầu, hãy yêu cầu AI bằng câu prompt: "Tạo 1 folder ở local làm nơi chứa agent".

3.2. Agent không hoạt động khi folder không nằm ở Local
Nếu để agent trong thư mục Downloads, agent có thể không hoạt động. Để các công cụ truy cập được vào folder của agent, bạn nên đặt nó ở một vị trí cố định trên máy (local) — ví dụ trong thư mục Documents hoặc một thư mục làm việc riêng — và chỉ rõ đường dẫn đó cho AI.
4. Công cụ & Extension cần cài
4.1. Cài extension Claude Code cho VSCode
Các IDE (phần mềm lập trình) cho phép cài thêm extension như Claude Code và Codex. Nếu muốn dùng Claude Code hoặc Codex ngay trong Visual Studio Code hay Antigravity, bạn chỉ cần cài extension tương ứng để build agent.
4.2. Yêu cầu update Node.js / Python đang chạy phiên bản cũ để chạy skill
Nếu máy bạn đang chạy Node.js hoặc Python phiên bản cũ, thì trong lúc chạy bộ skill AgentBase, Claude hoặc Codex có thể yêu cầu cập nhật lên phiên bản mới hơn. Lúc này bạn chỉ cần đồng ý (approve), Claude Code hoặc Codex sẽ tự động cập nhật lên phiên bản phù hợp rồi chạy tiếp.
4.3. Khi cài Docker gặp lỗi update WSL 
Sao chép dòng lệnh mà hệ thống báo, mở Terminal trên máy và dán lệnh vào để chạy. Nếu cài đặt Linux (WSL) thành công thì bạn có thể tiếp tục cài Docker.
Nếu gặp lỗi 403 Forbidden, nguyên nhân thường là tính năng ảo hóa (Hyper-V) trên máy chưa được bật. Hãy liên hệ IT Helpdesk để được hỗ trợ.
5. Luồng build Agent trên AgentBase
5.1. Đẩy agent rỗng lên AgentBase khi chạy skills mà chưa build agent
Bộ skill chạy theo quy trình 9 bước để triển khai agent lên AgentBase. Nếu bạn chạy hết quy trình này khi use case của agent chưa được xây dựng xong, agent vẫn được triển khai nhưng chưa có chức năng — tức là đã có runtime và phân quyền truy cập (access control), nhưng bản thân agent thì chưa làm được gì.
Vì vậy, nếu build agent từ link bộ skill ngay từ đầu, sau khi thêm credential xong (đây là bước 1 trong 9 bước lên AgentBase), bạn hãy DỪNG lại để xây dựng agent trước; build xong mới đi tiếp bước 2.
Một cách khác là dùng công cụ vibe code để build agent trước, sau đó mới import (hoặc clone) bộ skill AgentBase vào để chạy tiếp.
5.2. Không thể share link agent public được
Sau khi build và chạy agent trên máy (local) xong, khi đẩy agent lên runtime, đường link truy cập agent có thể vẫn trỏ về localhost (địa chỉ chỉ chạy được trên máy bạn). Vì vậy, mở link này trên trình duyệt khác hoặc gửi cho người khác sẽ báo lỗi "Not Found".
Trong trường hợp đó, bạn hãy thêm một câu prompt yêu cầu AI chuyển endpoint sang chế độ public, để có một đường link hoạt động được mà bạn có thể chia sẻ cho người khác.
Đây cũng chính là đường link để chia sẻ agent cho các Starter khác cùng trải nghiệm.
6. Lỗi khi nhập IAM Credential
6.1. IAM credential gặp lỗi sai/thiếu/không authen được
Khi nhập IAM credential mà hệ thống báo không xác thực (authenticate) được, hãy kiểm tra lại xem khi sao chép có bị thừa, thiếu ký tự hoặc dư khoảng trắng ở đầu/cuối hay không, rồi nhập lại.
7. Usecase Memory (lưu trữ memory)
7.1. Memory của agent lưu ở local thay vì được đẩy lên AgentBase
Khi mới build agent trên máy (local), các công cụ AI thường gợi ý lưu file memory.md ngay tại máy. Tuy nhiên, để agent chạy ổn định và ghi nhớ tốt, bạn nên đưa phần này lên tính năng Memory của AgentBase. Khi đó, memory được lưu tập trung, dài hạn và không bị mất khi bạn đổi máy.
Bạn có thể ra prompt yêu cầu agent đẩy memory từ máy lên module Memory của AgentBase; bộ skill sẽ tự thực hiện việc này.
